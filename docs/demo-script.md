# Synapse Demo 录制脚本

本脚本用于录制 2-3 分钟项目演示视频。建议录制时打开浏览器访问：

```text
http://localhost:5173/
```

## 0. 开场

讲解：

> 这是 Synapse AI 同传翻译助手。它不是单一翻译框，而是把语音同传、网页翻译和文本翻译整合到同一个 Web App 中。后端通过 DeepSeek API 完成翻译，API key 只保存在本地服务端，不暴露给浏览器。

镜头：

- 展示顶部三个入口：`语音同传`、`网页翻译`、`文本翻译`。
- 展示左侧 `英译中 / 中译英` 切换。

## 1. 文本翻译：中英夹杂到英文

操作：

1. 点击 `文本翻译`。
2. 切换到 `中译英`。
3. 输入：

```text
我们今天要讨论 cache policy 和边缘节点的 latency budget。
```

4. 点击 `开始翻译`。

讲解：

> 这里输入的是中英夹杂内容。模型会识别中文语义，同时保留技术术语，把整句统一翻成英文。

期望结果：

```text
Today we will discuss cache policy and the latency budget of edge nodes.
```

强调：

- 支持中文、英文、中英夹杂。
- 技术词不会被硬翻坏。
- 时间线里会显示延迟、置信度和说明。

## 2. 文本翻译：中英夹杂到中文

操作：

1. 切换到 `英译中`。
2. 输入：

```text
This architecture 会把 streaming ASR 和回改引擎结合起来。
```

3. 点击 `开始翻译`。

讲解：

> 这一句英文和中文混在一起，系统会统一输出成中文，并把 streaming ASR 这种技术词翻成更自然的中文表达。

期望结果：

```text
这种架构会把流式语音识别和回改引擎结合起来。
```

## 3. 网页翻译

操作：

1. 点击 `网页翻译`。
2. 输入：

```text
https://example.com/
```

3. 点击发送按钮。

讲解：

> 网页翻译会由后端抓取公开网页，抽取标题、段落和列表项，然后逐段调用 DeepSeek 翻译。前端不是等待整篇翻译结束，而是边返回边显示。

强调：

- 网页正文抽取在后端完成。
- 返回格式是 NDJSON 流。
- 适合英文文档、技术博客、课程页面快速阅读。

## 4. 语音同传

操作：

1. 点击 `语音同传`。
2. 选择 `英译中`。
3. 点击 `开始识别`。
4. 说：

```text
Today we will discuss the cache policy and latency budget for edge nodes.
```

讲解：

> 语音同传的链路是：浏览器先识别语音，后端再调用 DeepSeek 翻译，页面展示原文、译文、延迟和置信度。

可选：

- 打开 `语音播报`，展示翻译结果朗读。

## 5. 架构说明

讲解：

> 前端由 React 和 Vite 构建，后端是 Node + Express。前端只调用本地后端接口，不直接持有模型密钥。后端统一处理 `/api/translate` 和 `/api/webpage/translate-stream`，再路由到 DeepSeek。这样既保护密钥，也方便未来切换模型供应商。

展示 README 中架构图。

## 6. 结尾

讲解：

> 当前版本已经实现三个可用场景：语音同传、网页翻译、文本翻译。下一步可以继续扩展成浏览器插件、系统音频捕获、PDF 翻译和自定义术语库。
